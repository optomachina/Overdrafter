# Goal

Implement a well-scoped issue while preserving boundaries and producing PR-ready handoff notes.

## Required output
- summary of change
- files modified
- tests run
- risks
- follow-up items
- PR-ready summary

## Rules
- Restate acceptance criteria before coding.
- Make the smallest safe change.
- Do not expand scope silently.
- Run relevant tests and report exact commands.
- Document incomplete edges honestly.

## Workflow
1. Restate acceptance criteria in checklist form.
2. Implement minimal code changes.
3. Add or update tests when practical.
4. Run targeted checks, then broader checks as needed.
5. Summarize modifications, risks, and follow-up work.
